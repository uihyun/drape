# Drape — 기획 브리프

> 디지털 옷장 + OOTD 캘린더 + 룩북 SNS 앱. **가상 피팅을 핵심 차별점으로 미는** 옷장 앱.
> Claude Code와 첫 대화를 시작하기 위한 정리 문서.
> 기술 스택은 achelier.co (voda project)에서 대부분 복사해 따른다.

---

## 1. 한 줄 정의

**내 옷장을 통째로 디지털화하고, 직접 입어보지 않아도 내 모습에 입혀봐서, 매일 뭐 입을지 고민하는 시간을 없애주는 앱.**

옷 촬영 → 옷만 깔끔하게 크롭 → 자동 태깅 → 디지털 옷장 → 코디/캘린더 기록 → **내 사진에 입혀보기(가상 피팅)** → 사람들과 SNS처럼 공유.

## 2. 포지셔닝

인테리어 앱처럼 특정 상황에만 쓰는 게 아니라, **누구나 매일 옷을 입으니까 매일 여는 앱.** 옷장 뒤지는 물리적 수고 자체를 없앤다.

**핵심 차별점 = 가상 피팅.** 경쟁자(Lekondo)는 기록/룩북 중심. Drape는 "직접 안 입어봐도 미리 본다"를 메인으로 민다. 후속으로 머리 모양·립색·메이크업까지 바꿔보는 스타일 시뮬레이션으로 확장.

## 3. 경쟁 분석 — Lekondo (lekondo.com / lekondo.nyc)

이미 동일 컨셉으로 iOS·안드로이드 출시됨. 정면으로 따라가면 진다.
- 슬로건: *"Third space for fashion — Log your outfits. Build your closet. Explore the global lookbook."*
- 탭: **Outfits / Calendar / Closet** + SNS형 프로필(팔로워/팔로잉)
- **진짜 무기 = "Ontology"**: 36개+ 패션 미학(athleisure, gorpcore, Y2K, brutalism…)을 직접 정의한 분류 체계. 옷 태깅 + SEO 콘텐츠 + 룩북 탐색의 축으로 활용.

→ Drape는 **가상 피팅 품질**로 이긴다. 우리도 옷 태그 택사노미를 자산으로 만들되, 1번 무기는 "입혀보기".

## 4. 핵심 기능

### MVP
1. **Closet (옷장)** — 앱의 심장. 등록 마찰 최소화가 최우선.
   - 옷 촬영 → 배경 제거로 옷만 크롭 (입은 사진을 찍어도 옷만 추출)
   - 자동 태깅(카테고리/색/계절/스타일) + 수동 보정
   - 검색/필터로 내 옷 빠르게 찾기
2. **Virtual Try-On (가상 피팅)** — MVP에 포함. 핵심 차별점.
   - 내 전신 사진(정체성 레퍼런스)에 옷장 아이템 선택해 입혀보기
   - 코디 전체(상의+하의+아우터) 합성
3. **Outfit (코디 / OOTD)** — 옷장 아이템으로 코디 조합·저장
4. **Calendar (Daily Lookbook)** — OOTD 월별 캘린더 기록
5. **Feed / SNS** — 공개 프로필, 팔로우, 코디 세이브·좋아요·댓글

### Phase 2
6. **스타일 시뮬레이션** — 머리 모양·립색·메이크업 바꿔보기 (가상 피팅과 같은 모델로 처리)
7. **장소 지정 미리보기** — 특정 장소/TPO에서 이 코디의 느낌
8. **커플/친구 매칭** — 서로 옷장 공유, 매칭 코디 추천, 상대 옷 입혀보기

### Later
9. 옷 당근식 교환/판매
10. 상품 사진 찍으면 비슷한 옷 추천·검색

## 5. 기술 의사결정 (확정)

스택은 voda project 복사. 신규로 붙는 외부 기능만 정리.

| 기능 | 결론 |
|---|---|
| 가상 피팅 / 헤어·립·메이크업 / 옷만 크롭 / 배경 변경 | **Google Gemini — Nano Banana Pro (Gemini 3 Pro Image)** 단일 모델. 범용 편집이라 피팅·헤어·립·메이크업·장소 분위기까지 한 모델로 다 됨. |
| 옷 자동 태깅 (vision) | **Gemini (3 Pro / Flash) vision.** 같은 벤더로 통합 → SDK·계정·청구 하나. 옷의 카테고리/색/스타일 구조화 추출. |
| 빠른/저렴한 변형 | 어려운 케이스(정체성 보존 중요)는 **Pro**, 가벼운 변형·미리보기는 **Nano Banana 2 (Gemini 3.1 Flash Image)** 로 라우팅. |
| (선택) 배경제거 비용 최적화 | 대량 누끼 비용이 부담되면 PhotoRoom Remove BG($0.02/장)를 옷 크롭에만 끼우는 옵션. 기본은 Gemini 단일 유지. |

**왜 Gemini 단일인가:** 로드맵 전체(피팅 → 헤어/립/메이크업 → 장소)를 한 API로 커버. Nano Banana Pro는 정체성 보존이 간판 기능 — 최대 14장 입력 / 최대 5명 정체성 유지. 가격도 합리적(2.5 Flash 기준 ~$0.039/장). **3D는 MVP 제외**(등록 마찰 때문).

## 6. 옷 등록 = 비동기 파이프라인 (등록 속도가 생사를 가른다)

"옷 한 벌 추가"가 체감 3초 안에 끝나야 함. 그래야 옷장이 채워지고, 옷장이 채워져야 나머지 기능이 의미를 가진다.

**플로우:**
1. 촬영 즉시 옷장에 **placeholder 카드(스켈레톤)** 추가 → 사용자는 바로 다음 옷 촬영 가능
2. 백그라운드 작업 큐에 enqueue
3. 워커가 비동기로: 배경제거 크롭 → 자동 태깅 → 결과 저장
4. 끝나면 그 카드만 `processing → ready` 로 토글 (실시간 업데이트)

> Firebase면 Cloud Functions + Firestore 실시간 리스너로 자연스럽게 구현. 사용자는 처리 끝날 때까지 절대 안 기다린다.

## 7. 가상 피팅 품질 — 처음부터 제대로 (어색한 케이스 0 목표)

핏보다 **캐릭터(정체성) 일관성**이 우선. 처음부터 아래를 다 갖춰서 시작한다.

**모델 밖 파이프라인으로 품질 확보:**
- **정체성 레퍼런스 사진**: 사용자별로 전신 2~3장(여러 각도)을 저장 → 매 생성마다 같이 입력. Nano Banana Pro의 멀티 이미지 일관성 활용.
- **입력 품질이 80%**: 깔끔한 전신, 일관된 포즈/조명, 단순 배경. → **사용자 가이드 필수**(아래).
- **프롬프트 고정**: "얼굴·체형·포즈 보존, 옷 핏/드레이프는 이렇게" 명시적 지시. 정체성 보존을 매 호출에 강제.
- **후처리 face paste-back**: 생성 결과에 원본 얼굴을 다시 합성해 정체성 락. (정체성 보존 프로 트릭)
- **N개 생성 → 베스트 선택**: 여러 장 뽑아 품질 스코어/사용자 선택으로 최적 채택.
- **모델 티어 라우팅**: 정체성 중요·복잡한 합성 → Pro / 가벼운 변형 → Flash.

**사용자 입력 가이드(온보딩 + 촬영 화면 오버레이):**
- "전신이 다 나오게, 정면, 밝은 곳에서"
- "단색/단순 배경 추천"
- 좋은 예/나쁜 예 샘플 이미지 제시
- 정체성 레퍼런스 등록을 첫 온보딩에 자연스럽게 포함

## 8. 피드백 수집 루프 (1일차부터 — "못 고쳐도 데이터는 모은다")

닫힌 API라 지금은 모델 자체를 파인튜닝 못 함. 그래도 **피드백은 처음부터 모은다.**

- 모든 생성 결과에 **👍 / 👎 + "다시 생성"** 버튼
- 어떤 입력/프롬프트/모델 티어가 좋은 결과를 냈는지 신호 저장 (input 사진 메타, prompt, model, 결과, rating)
- 이 데이터의 가치: (a) 지금은 프롬프트·파이프라인 튜닝 연료, (b) 나중에 오픈소스 try-on(IDM-VTON / CatVTON / Leffa 등) 자체 호스팅으로 갈 때의 **학습 데이터**.
- 파인튜닝을 못 한다고 피드백을 안 모으면, 나중에 가고 싶어도 자산이 없다. → **무조건 1일차부터.**

## 9. 데이터 모델 (초안)

- **User**: 프로필, 팔로우 관계, **정체성 레퍼런스 사진(2~3장)**
- **Item (옷)**: 원본/크롭 이미지, 카테고리, 색, 계절, 브랜드, 태그[], `status(processing/ready)`
- **Outfit (코디)**: Item[] 조합, 태그
- **OOTD**: 날짜, Outfit 또는 사진, 메모 → Calendar에서 렌더
- **Tag**: 우리 택사노미(카테고리/색/스타일)
- **Generation (가상 피팅 결과)**: 입력 사진 ref, 입힌 Item[], prompt, model, 결과 이미지, **rating(👍/👎)**, 재생성 여부 → 피드백 루프의 핵심 테이블

## 10. 디자인 방향

- Lekondo처럼 **깔끔한 모노톤 + 작은 포인트 컬러**, 에디토리얼 톤.
- 크롭된 옷 누끼 / 피팅 결과가 주인공이 되도록 여백 넉넉히.
- 메인 네비: **Closet / Outfit(OOTD) / Calendar(Daily Lookbook)** + 피드/프로필.

## 11. 기존 자산 재활용

- **achelier.co (voda project)** 의 스택·인증·DB·SNS(피드/좋아요/댓글/팔로우) 구조 복사.
- 신규: 카메라 촬영 플로우, 비동기 등록 파이프라인, Gemini 연동(피팅+태깅), 정체성 레퍼런스 관리, 피드백 루프, 캘린더 뷰.
